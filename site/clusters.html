Cluster labeling

Group representatives from HDBSCAN clustering of the 350 confirmed SRKW keeps. Each card represents one acoustic cluster — listen to the representative (and sibling examples if helpful), assign a Ford call code (or your best guess). Labels propagate to all cluster members when the classifier is trained next.
How to use this page
  • For each cluster, click play on the representative (the clip closest to cluster centroid — most "typical" of the cluster).
  • If you want a second opinion, the siblings are the next 4 nearest clips in the same cluster — they should sound similar.
  • Type a Ford call code in the label box (e.g. S01, S17, S37ii) or one of the special values from the dropdown: unk (you can't tell), not-stereotyped.
  • If a cluster contains more than one call type (e.g. the rep is S01 but several siblings are S17), enter the labels comma-separated: S01,S17. The classifier will learn it as a multi-label cluster. Or use multiple if you can't disentangle them.
  • It's fine if multiple clusters get the same label (e.g. three clusters all sound like S01). HDBSCAN groups by acoustic similarity, which can split a call type into recording-condition / individual / intensity variants. Same-label clusters help the classifier learn the variation.
  • Labels auto-save to your browser's localStorage as you type. Click Export labels in the top bar to download a JSON file — that's what feeds the classifier next.
  • Reference: Orcasound's SRKW call catalogue.
